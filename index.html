Layer 1 — camera (angle + speed)
How does the camera detect needle movement?
Webcam / phone cam
Films the needle 30 times per second
→
ArUco marker
Tiny printed sticker on needle hub
→
OpenCV detection
Finds marker in each frame
→
Pose estimation
Calculates 3D rotation + position
→
Angle + speed
Final values sent to app
↓
Challenge — ArUco marker visibility
Problem identified
Why does the marker become invisible during real insertion?
Medical grip
Full hand wraps around catheter hub
→
Marker blocked
Hub face hidden under fingers
→
Tracking lost
OpenCV cannot detect → no angle / speed
↓
Solution — vertical axis extension arm
3D printed fix
How does the vertical arm keep the marker always visible?
Mating connector
Snap-fits onto injection port, wings stabilize
→
Vertical post ×2
Rises 30mm above hand grip zone
→
Elbow joint
Offsets marker to face camera directly
→
ArUco disc platform
Perpendicular to camera — always detectable
Material — TPU 95A Post height — ~30mm Platform — Ø24mm Mount — snap-fit Cost — ~$2

Layer 2 — electric circuit (depth)
How does the prop know how deep the needle went?
5V power
From ESP32 pin — always on
→
10kΩ resistors ×4
One per depth layer — limits current
→
Copper strips in prop
4 layers: 2mm / 5mm / 8mm / 11mm
↕
Metal needle
Wire clipped to hub → GND
→
ESP32 analog pins
A0–A3 reads voltage drop
↓
Layer 3 — WiFi data transfer
How does data get from the prop to the phone?
ESP32 (main chip)
Runs circuit + sends WiFi
→
JSON packet
depth, angle, speed, status
→
WiFi router
Local network bridge
→
WebSocket
Live connection every 100ms
→
App (React Native)
Receives + displays data
↓
